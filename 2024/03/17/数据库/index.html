数据库

2024年03月17日

数据库

一、数据库三大范式

第一范式（1NF）：属性不可分割（表中的每一列都可以满足需求，不必再拆分）

第二范式（2NF）：在第一范式的前提下，非主键完全依赖于主键，在联合主键的情况下，非主键不能部分依赖于主键。

第三范式（3NF）：在第二范式的前提下，不存在传递依赖的关系（即不能A->B,B->C，导致间接依赖）。

非主键列越少的表越容易满足3NF。

二、sql语句

DDL（definition）：定义数据，定义数据库对象（库，表，列）。

DML（manipulation）:操作数据，对数据进行增删改。

DCL（control）：控制数据，定义访问权限和安全级别。

DQL（query）：查询数据。

三、DDL操作数据库

用cmd打开D:\mysql\mysql-8.0.28-winx64\bin

mysql -u root -p//登录数据库

create database 数据库名；//创建新的数据库

show databases;//查看所有数据库

alter database 数据库名 character set 编码方式；//修改数据库（数据库中存储数据所使用的字符集编码方式，例如从 utf8 修改为 utf8mb4。）

drop database 数据库名；//删除数据库

select database();//查看当前使用的数据库

NULL表示暂未连数据库

use 数据库名；//连接这一数据库；

create table 表名（列名1 数据类型 [约束]，列名2 数据类型 [约束]，列名3 数据类型 [约束]）；

*[约束]表示可有可无

数据类型：

int;

double;(如double(5,2)为最多5位数，但有2位为小数，最大为999.99)

char;(固定字符串长度，如char(10)‘bbb’，因为固定占10位)

varchar;(可变字符串长度,如varchar(10)’bbb’，因为可变占3位)

test;(字符串类型，如小说信息)

blob;(字节类型，如视频、音频、图片)

date;(日期类型，年、月、日)

time;(时间类型，时、分、秒)

timestamp;(时间戳类型，年月日时分秒，会自动赋值)

datatime;(日期时间类型，年月日时分秒)

———-约束
         定义：表中数据的限制条件（保证表中记录完整有效）
1、非空约束（not null）

必须给定具体的数据进行约束

如name varchar(10) not null,

2、唯一性约束（unique）

不可重复，但可以为空null

如email varchar(32) unique,

表级约束：email varchar(32),

                unique(email)

                注：如果插入相同emial会报错

用表级约束给多个字段联合约束：

               name varchar(10) not null,

               email varchar(32),

               unique(name,email)

               注：两个或两个以上字段与已存入的数据相同则报错

用表级约束给约束起名字：

               constraint other\_name unique(email)

               constraint(约束关键字),other\_name自我取名

     例：name varchar(32) not null unique(名既不能空，也不能重复)
3、主键约束（primary key)

添加主键约束，会将该字段设为主键字段，主键字段中的每一个值称为主键值。

主键约束除了字段不能空不能重复以外，还会添加索引——index（提高查找效率）。

一张表主键约束只能有一个，但可以作用于好几个字段。

单一主键：id int(10) primary key,（列级定义）/

               id int(10),

               constraint other\_name primary key(id),(表级定义)

复合主键：id int(10),

                name varchar(32) not null,

                email varchar(128) unique,

                 primary key(id,name),(表级定义)

mysql提供一个自增数，自动生成主键值，从1开始递增（auto_increment)

               例：id int(10) primary key auto\_increment,
4.外键约束（foreign key）

外键：一个字段是A的主键，也是B的字段，那么B是该字段的外键。外键用来维护两个表的一致性。

添加外键约束，会将该字段设为外键字段，外键字段中的每一个值称为外键值。（一张表可以有多个外键约束。与主键不同）

注：1.外键值可以为空

    2.外键字段引用另一张表的字段时，被引用字段必须有unique约束

    3.有了外键引用后，表分为父子两表

    4.先创建父表，先删除子表数据，先插入父表数据

show tables;//查看所有表

desc 表名;//查看列字段信息

drop table 表名;//删除表

alter table 表名 add 新列名 新的数据类型；//添加列

alter table 表名 change 旧列名 新列名 新的数据类型；//修改列

alter table 表名 drop 列名；//删除列

alter table 旧表名 rename 新表名；//修改表名（注：不能desc student;）

show create table 表名；//查看表格的创建细节

alter table 表名 character set 编码方式;//修改字符集

四、DML操作数据库

1.插入（insert）

insert into 表名(列名) values(数据值);

注：多个列名与多个列值之间用，隔开；列名与列值要一一对应；非数值的列值两端要加’ ‘。

insert into student(name,age,sex) values(‘小白’,19,’g’);

若给所有列添加数据，可省略列名，但要按照顺序添加值。

insert into student values(‘小白’,19,’g’);

也可以同时添加多行；

insert into student(name,sex,age) values(‘Yoky’,’g’,19),(‘Yohy’,’b’,19),(‘woky’,’b’,19);

select *from 表名；//查询所有数据

2.修改/更新（update）

update 表名 set 列名1=列值1，列名2=列值2…;（全改）

若限定某一值改：update 表名 set 列名1=列值1 where 列名=值；

改一行多列：update student set 列名1=列值1，列值2=列值2… where 列名=列值；

可以给某一数值+、-、*、/、%

例如：给Yohy的年龄加3

3.删除（delete/truncate)

delete from 表名【where 列名=值】

delete from 表名（注：删除了表中信息，但表仍然存在）

truncate（数据找不回）不像delete（数据可找回）一样只删除表中数据，它是删除整个表，再创建一个相同的新表。

五、DCL操作数据库

设置或更改数据库用户或角色权限的语句。

创建用户：

        一.create user 用户名@指定ip identified by 密码;

        create user yhpa@localhost identified by 'abc';

        二.create user 用户名@客户端ip identified by 密码;

        只有指定ip能登录

        三.create user 用户名@'%' by 密码;

        任意ip均可登录

给用户授权：

        一.指定授权

        grant 权限1，权限2...... on 数据库名.\* to 用户名@IP;

        数据库名.\*指的是这个数据库中的所有表

        二.授予所有权限

        grant all on \*. \* to 用户名@IP；

查询用户权限：

        show grants for 用户名@IP;

撤销用户权限：

        revoke 权限1，权限2...... on 数据库名.\* from 用户名@IP;

删除用户：

        drop user 用户名@IP；

退出用户：

         quit;

六、DQL基础操作数据库

DQL查询是让数据库发送结果集给客户端，返回的是一张虚拟表，不会改变数据库中的值。

语法：select 列名 from 表名【限定】

限定：where(限定行)

        group by grouping \_columns(对结果分组)

        having condition(分组后行条件)

        order by sorting\_columns(对结果分组)

        limit offset\_star,row\_count(结果限定)
  1. select *from 表名；//查询所有列
  2. select 列名1，列名2…… from 表名；//指定查询列
sql逻辑运算符：

并且：and；或者：or；取非：not

  1. select *from 表名 where 限定语句;
    并且用and

或者用or(注：用or查询相同列的两个数据不能省略列名)

查找同列名可写为：列名 in(列值1，列值2)

查找不是同列名的信息：列名 not in(列值1，列值2)

查询列值为空：列名 is null

查询列值在范围内：列名 between 最小值 and 最大值

查询非某项列值：！或者<>

七、模糊查询（DQL）

类似于关键词查询

语法：列名 like ‘表达式’//表达式为字符串

用通配符替代不知道的信息，_代表任意一个字符，%代表任意一个或多个字符。

查询名为三个字符：select *from stud where sname like ‘___‘;(3个下划线)

查询名由5个字符组成，第5个字符为‘i’：select *from stud where sname like ‘i’;(4个下划线)

查询名以’z‘开头：select *from stud where sname like ‘z%’;

查询名第二个字符为’i’：select *from stud where sname like ‘_i%’;

查询名中包含‘a’：select *from stud where sname like ‘%a%’;

八、字段控制查询

1.去除重复记录

2.查询列的和

注：必须两列可以相加。

两列中任何一列为null就会导致相加结果为null，所以要将null变为0，用ifnull函数。

select 列名+ifnull(要转换的列名,0) from 表名；

3.给列取别名

（1)select 要改的列名 as 新列名 from 表名;

（2)select 要改的列名 新列名 from 表名；（将as改为空格）

九、排序

select *from 表名 order by 列名 asc/desc;

asc:升序；desc:降序（不写默认升序）

多列排名：当前一个列值相同，再进行后一个列值比较。

select *from 表名 order by 列名1 排序方式，列名2 排序方式；

十、聚合函数

定义：做纵向运算的函数

count(列名):统计该列不为空的行数；

max(列名)：记录该列的最大值（字符串类型按字符串排序运算）；

min(列名)：记录该列的最小值（字符串类型按字符串排序运算）；

sum(列名)：计算该列的数值和，如果不是数值类型，结果为0；

avg(列名)：计算该列的平均值，如果不是数值类型，结果为0；

一、count

  1. select count(*) from 表名;(查看表中所有行数)
  2. select count(列名) from 表名;(查看表中该列非空行数)
  3. select count(*) from 表名 where 限制条件;(查看表中满足一定条件的行数)
    可以以和为限制，不一定非为表中已有列。
  4. select count(列名1),count(列名2) from 表名;(同时列出多个列的非空行数)

二、sum、avg和min、max

  1. 和：select sum(列名) from 表名;

2.平均值：select avg(列名) from 表名；

3.最大最小值：select max(列名),min(列名) from 表名;

十一、分组查询

查询语句中有分组情况，那么select后只能是聚合函数和被分组的列名。

1.表中行全部分组

select 方法（列名） from 表名 group by 要分类的列名；

分类一一对应：select 要分类的列名,方法（列名） from 表名 group by 要分类的列名；

2.分类中加限定（先判断条件再分类）

select 要分类的列名,方式（列名） from 表名 where 限定条件 group by 要分类的列名；

十二、HAVING子句（先分类再判断条件）

select 要分类的列名,方式（列名） from 表名 group by 要分类的列名 having 方式（列名）的限定条件;

having和where的区别：

  1. having在分组后对数据进行筛选，而where实在分组前对数据进行。
  2. having后可以使用分组函数，但where不行。（如果没有满足条件的数据，where就不会执行）

十三、limit关键字和分页查询

select *from 表名 limit 开始行，查询行数；
